一、限时折扣促销
	参与者：商户、住户
	
	价值描述：对商户，卖得更快
	对于商户中快要到期的商品，商户需要快速将东西卖出去，这个诉求应该比较强烈
（需要确认：是否商户可以将卖不出去的货品原价退还给批发商？）快过期的东西如果
不早点卖掉，过期了就浪费了，所以对小区商户来讲诉求是有的，特别是对于蔬菜等商
品。
	对于畅销商品，可以采取薄利多销的方式，赚得更多。
	
	价值描述：对住户，买得更方便
	大多数用户是喜好打折的东西的，能通过便条网，看到便宜的东西，并在商户间做
对比，这比自己下楼买要方便。
	
	相对传统方式的排它性优势：
	传统方式，商户仅能把打折信息告诉到店里来的住户，现在还能通过网络告知，
东西会卖得更快。
	传统方式，住户必须到商户店里才能选东西，现在可以足不出户选择，若商户能送
货就更方便。
	弱势：无，对传统方式是个补充
	如何弥补：不涉及
	
	相比同行的排它性优势：良好的信用关系
	小区商户相比网上商户卖食品来说，更加值得信赖。因为住户经常到小区商户内买
东西，他们对小区的商户是认可的，而一般的网上商店，很难建立起这种信用关系。
	弱势：店的销售量小，店的客户少
	如何弥补：增加便条网小区的入住率
	
	步骤：
	1. 商户：登录便条网，进入如自己的商铺店面、拍照商品的一组照片、设置促销
	价格、时限等信息，并发布
	2. 住户：访问便条网，默认来到自己的所属小区位置、点击“小区内促销商品”，浏
	览、并选择自己想要商品
	3. 住户-商户：住户发实时消息到商户（可以是语音/文字），询问送货时间，反复
	交流协商时间、价钱等，记录送货地点。
		若此商品不支持送货，过程终止，用户自行到商品所在商户那里购买
		若协商不成，过程终止
	4. 商户送货员：进入“待送货清单”，逐个送货、收钱，完成交易。商户向住户询问
	评价信息，通过手机实时评价。（系统要能自动合理的分类送货单，指导送货员高效
	地送货）
